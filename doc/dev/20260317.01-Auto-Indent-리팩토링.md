# Auto Indent 리팩토링 정리

## 개요
`auto` 들여쓰기 모드를 `formatter.format()` 외부에서 후처리하던 방식을 내부로 통합하여 리팩토링

## 기존 구조의 문제점

### 1. 이중 처리 문제
- `formatter.format()`에서 기본 들여쓰기 처리
- `applyAutoIndent()`에서 다시 들여쓰기 재처리
- 동일한 SQL에 대해 두 번의 포매팅 과정 수행

### 2. 설정 충돌
- `formatter.format()`: `spaces`/`tabs` 옵션 사용
- `applyAutoIndent()`: 항상 `spaces`로 고정
- 사용자 설정과 실제 동작 불일치

### 3. 비효율성
- 두 번의 파싱/AST 생성 과정
- 불필요한 문자열 처리 반복
- 코드 복잡성 증가

## 리팩토링 내용

### 1. FormatOptions 타입 확장

  dialect?: SqlDialect
  templateType?: 'none' | 'mybatis'

format(sql: string, options: FormatOptions = {}): string {
  const cfg = { ...this.config, ...options }
  
  // ... MyBatis 처리

    const tokens = this.tokenizer.tokenize(workSql)

    if (options.autoIndent) {
      result = applyAutoIndent(result, cfg)
    }

    // ... 줄바꿈 정규화, MyBatis 복원
    
    return result

}

### 3. formatSql.ts 단순화
```typescript
// src/utils/formatSql.ts
function buildFormatOptions(rules: FormatRulesState, detectedDialect?: SqlDialect) {
  return {
    dialect: 'sql' as const,
    tabWidth: rules.indentEnabled ? rules.tabWidth : 2,
    indentType: (
      rules.indentEnabled && rules.indentType === 'tabs'
        ? 'tabs'
        : 'spaces'
    ) as IndentType,
    keywordCase: rules.keywordCaseEnabled ? rules.keywordCase : ('preserve' as const),
    denseOperators: rules.operatorSpacingEnabled ? rules.denseOperators : false,
    commaPosition: (rules.commaPositionEnabled ? rules.commaPosition : 'trailing') as CommaPosition,
    templateType: (detectedDialect === 'mybatis' ? 'mybatis' : 'none') as 'mybatis' | 'none',
    // ✅ autoIndent 플래그만 전달
    autoIndent: rules.indentEnabled && rules.indentType === 'auto',
  }
}

export function formatWithRules(sql: string, rules: FormatRulesState): FormatResult {
  const formatter = new SqlFormatter()
  const detectedDialect = formatter.detectDialect(sql)
  const options = buildFormatOptions(rules, detectedDialect)
  const formattedSql = formatter.format(sql, options)  // ✅ 단일 호출

  return {
    sql: formattedSql,
    detectedDialect,
    detectedDialectLabel: DIALECT_LABEL[detectedDialect],
  }
}
```

## 개선 효과

### 1. 단일 책임 원칙
- `formatter.format()`: 모든 포매팅 로직 책임
- `formatSql.ts`: 옵션 변환 및 호출만 담당

### 2. 일관성 확보
- 설정이 한 번만 적용되어 충돌 없음
- `autoIndent` 플래그로 명확한 제어

### 3. 성능 향상
- 한 번의 파싱으로 모든 처리 완료
- 불필요한 후처리 제거

- `formatSql.ts`: 79라인 → 55라인 (-30%)
- 복잡한 `FormatterConfig` 재구성 제거
- 명확한 인터페이스 설계
