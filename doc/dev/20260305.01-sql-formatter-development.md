# SQL Formatter 개발 문서
### 0. Windsurf
- **SWE-1.5 모델 사용**

## 개요

sql-formatter 라이브러리 의존성을 제거하고 자체 SQL 포매터를 개발하는 프로젝트

## 현재 진행 상태

### ✅ 완료된 작업

#### 1. 아키텍처 설계
- **토크나이저**: SQL 문장을 토큰 단위로 분리
- **파서**: 토큰을 AST(Abstract Syntax Tree)로 변환
- **포매터**: AST를 기반으로 SQL 포매팅 적용
- **타입 시스템**: TokenType, FormatterConfig 등 정의

#### 2. 코어 구현
```
src/lib/sql-formatter/
├── core/
│   ├── tokenizer.ts     # 토큰화 구현
│   ├── parser.ts        # AST 파싱 구현
│   └── formatter.ts     # 포매팅 엔진
├── types/
│   ├── token.ts         # 토큰 관련 타입
│   └── config.ts       # 설정 관련 타입
└── index.ts           # 메인 진입점
```

#### 3. 기본 기능 구현
- **SQL 토큰화**: 키워드, 식별자, 연산자, 문자열 등
- **SELECT 문 파싱**: SELECT, FROM, WHERE 절 처리
- **기본 포매팅**: 키워드 대소문자, 들여쓰기, 콤마 위치
- **방언 감지**: PL/SQL, MySQL, PostgreSQL, T-SQL 식별

#### 4. 기존 시스템 연동
- **호환성 래퍼**: `formatWithRules()` 함수 유지
- **타입 변환**: `FormatRulesState` → `FormatterConfig`
- **에러 핸들링**: 포매팅 실패 시 원본 반환

### 🚧 현재 구현된 기능

#### SQL 토크화
- 키워드: SELECT, INSERT, UPDATE, DELETE 등
- 식별자: 테이블명, 컬럼명, 변수명
- 리터럴: 문자열, 숫자
- 연산자: =, <>, +, -, *, / 등
- 구분자: 콤마, 세미콜, 괄호 등
- 주석: `--`, `/* */`

#### SELECT 문 포매팅
```sql
-- 입력
select id, name, email from users where active = 1

-- 출력
SELECT
  id,
  name,
  email
FROM
  users
WHERE
  active = 1
```

#### 포매팅 규칙
- **키워드 대소문자**: upper/lower/preserve
- **들여쓰기**: spaces/tabs, 너비 설정
- **콤마 위치**: leading/trailing
- **연산자 공백**: dense/normal

#### 방언 감지
- **PL/SQL**: DECLARE, BEGIN, END, PROCEDURE 등
- **MySQL**: LIMIT, AUTO_INCREMENT, TINYINT 등
- **PostgreSQL**: SERIAL, JSONB, ILIKE 등
- **T-SQL**: TOP, IDENTITY, GETDATE() 등

### 📋 테스트 케이스

#### 기본 SQL
```typescript
const result = formatWithRules('select id, name from users', testRules)
// 결과: SELECT\n  id,\n  name\nFROM\n  users
```

#### PL/SQL
```typescript
const result = formatPlSql('declare v_name varchar2(100); begin select name into v_name from users; end;', testRules)
// 결과: DECLARE\n  v_name VARCHAR2(100);\nBEGIN\n  SELECT\n    name\n  INTO\n    v_name\n  FROM\n    users;\nEND;
```

## 🔄 다음 단계 계획

### 1. 포매팅 규칙 확장 (우선순위: 높음)

#### GROUP BY 절
```sql
-- 입력
select department, count(*) from employees group by department

-- 예상 출력
SELECT
  department,
  COUNT(*)
FROM
  employees
GROUP BY
  department
```

#### ORDER BY 절
```sql
-- 입력
select name, age from users order by age desc, name asc

-- 예상 출력
SELECT
  name,
  age
FROM
  users
ORDER BY
  age DESC,
  name ASC
```

#### JOIN 처리
```sql
-- 입력
select u.id, p.title from users u join posts p on u.id = p.user_id

-- 예상 출력
SELECT
  u.id,
  p.title
FROM
  users u
  JOIN posts p ON u.id = p.user_id
```

### 2. SQL 방언별 핸들러 (우선순위: 중간)

#### PL/SQL 전용 문법
- **블록 구조**: DECLARE-BEGIN-END
- **제어문**: IF-THEN-ELSE, LOOP, WHILE
- **예외 처리**: EXCEPTION, WHEN-THEN
- **커서**: CURSOR, OPEN, FETCH, CLOSE
- **프로시저/함수**: CREATE PROCEDURE, FUNCTION

#### MySQL 전용 문법
- **데이터 타입**: TINYINT, ENUM, SET
- **함수**: NOW(), DATE_FORMAT()
- **제한**: LIMIT, OFFSET

### 3. MyBatis 템플릿 파서 (우선순위: 중간)

#### XML 태그 처리
```xml
<!-- 입력 -->
<select id="getUser" resultType="User">
  SELECT * FROM user WHERE id = #{id} AND status = ${status}
</select>

<!-- 예상 출력 -->
SELECT
  *
FROM
  user
WHERE
  id = ?
  AND status = status
```

#### 파라미터 치환
- `#{param}` → `?` (PreparedStatement)
- `${param}` → `param` (직접 치환)

## 📝 구현 노트

### 현재 제한사항
1. **GROUP BY/ORDER BY**: 아직 구현되지 않음
2. **복잡한 JOIN**: 여러 JOIN, 서브쿼리 미지원
3. **PL/SQL 문법**: 블록 구조만 기본 지원
4. **MyBatis 템플릿**: XML 파싱 미구현

### 기술적 도전 과제
1. **AST 설계**: SQL의 복잡한 문법 구조 표현
2. **방언 호환성**: 각 DBMS의 고유 문법 처리
3. **성능 최적화**: 대용량 SQL 처리
4. **에러 처리**: 문법 오류의 graceful handling

### 해결 방안
1. **점진적 구현**: 기본 기능부터 단계별 확장
2. **플러그인 아키텍처**: 방언별로 모듈화
3. **테스트 주도 개발**: 각 기능별 단위 테스트
4. **문서화**: API 및 사용법 명확화
