# 서브쿼리 처리 로직 리팩토링

### 0. Windsurf
- **SWE-1.5 모델 사용**

## 개요
`applyAutoIndent` 안에만 존재하던 서브쿼리 대응 로직을 분리하여 `autoIndent=false`일 때도 서브쿼리가 올바르게 처리되도록 개선

## 기존 구조의 문제점

### 1. 서브쿼리 처리 로직의 편중
- 서브쿼리 추출/복원 로직이 `autoIndentFormatter.ts`에만 존재
- 기본 포매터(`formatter.ts`)는 단순 재귀 호출만 수행
- `autoIndent=false`일 때 서브쿼리 처리가 부정확함

### 2. 코드 중복 및 유지보수 문제
- 서브쿼리 관련 로직이 여러 곳에 흩어져 있음
- 서브쿼리 처리 로직 수정 시 두 파일 모두 변경 필요
- 별칭(alias) 처리가 일관되지 않음

### 3. 기능 제약
- 중첩 서브쿼리 처리가 제한적
- 서브쿼리 포매팅 옵션 부족
- 복잡한 서브쿼리 구조에서 들여쓰기 불일치

## 리팩토링 내용

### 1. 서브쿼리 처리 유틸리티 분리

#### 파일 생성: `subqueryUtils.ts`
```typescript
// 서브쿼리 관련 인터페이스 및 함수 분리
export interface SubBlock {
  key: string
  inner: string
  alias: string
}

export interface ExtractResult {
  sql: string
  subBlocks: SubBlock[]
}

// 핵심 함수들
export function extractSubqueries(sql: string): ExtractResult
export function restoreSubqueries(
  sql: string, 
  subBlocks: SubBlock[], 
  cfg: FormatterConfig,
  formatInner?: (inner: string) => string

```

### 2. formatter.ts의 formatSubquery 메서드 강화

#### 개선된 방식
```typescript
private formatSubquery(node: AstNode, cfg: FormatterConfig, indent: number): string {
  const selNode = node.children?.find(c => c.type === 'select_statement')
  if (selNode) {
    // 1. 서브쿼리 내용을 기본 포매팅으로 평탄화
    let flattened = this.formatSelectStatement(selNode, cfg, 0)
    
    // 2. 키워드 기준으로 미리 줄바꿈 적용
    flattened = presplitByKeywords(flattened, cfg)
    
    // 3. 서브쿼리 처리 유틸리티 적용
    const extracted = extractSubqueries(flattened)
    if (extracted.subBlocks.length > 0) {
      // 중첩 서브쿼리 재귀 처리
      const processed = restoreSubqueries(extracted.sql, extracted.subBlocks, cfg, ...)
      // 들여쓰기 적용...
    }
    // ...
  }
}
```

### 3. autoIndentFormatter.ts 단순화

#### 제거된 내용
- `extractSubqueries` 함수 (subqueryUtils로 이동)
- `restoreSubqueries` 로직 (subqueryUtils로 이동)
- `presplitByKeywords` 함수 (subqueryUtils로 이동)
- 서브쿼리 관련 인터페이스 정의

#### 남은 기능
- 키워드 정렬 및 패딩 로직
- 세그먼트 분류 및 병합
- Auto-indent 특화 기능

#### 개선된 구조
```typescript
// 유틸리티에서 import
import { extractSubqueries, restoreSubqueries, presplitByKeywords, ... } from './subqueryUtils'

function formatBlock(sql: string, cfg: FormatterConfig, baseIndent: string): string {
  // 1. 서브쿼리 추출 (유틸리티 사용)
  const extracted = extractSubqueries(sql)
  
  // 2. 키워드 정렬 처리 (기존 로직 유지)
  // ...
  
  // 3. 서브쿼리 복원 (유틸리티 사용)
  result = restoreSubqueries(result, extracted.subBlocks, cfg, (inner) => {
    const normalizedInner = presplitByKeywords(inner, cfg)
    return formatBlock(normalizedInner, cfg, baseIndent)
  })
}
```

## 개선 효과

### 1. 일관된 서브쿼리 처리
- `autoIndent` 옵션과 관계없이 모든 서브쿼리가 올바르게 처리
- 별칭(alias) 처리가 일관성 확보
- 중첩 서브쿼리 재귀 처리 개선

### 2. 코드 중복 제거
- 서브쿼리 로직이 한 곳(`subqueryUtils.ts`)에만 존재
- 유지보수성 향상: 수정 시 한 파일만 변경
- 테스트 용이성: 서브쿼리 로직 단독 테스트 가능

### 3. 확장성 확보
- 새로운 서브쿼리 포매팅 옵션 추가 용이
- 다른 포매터에서 서브쿼리 유틸리티 재사용 가능
- 플러그인 방식 확장 지원

### 4. 성능 최적화
- 불필요한 중복 처리 제거
- 효율적인 서브쿼리 추출/복원 알고리즘

## 테스트 권장사항

### 1. 기본 기능 테스트
```sql
-- 단순 서브쿼리
SELECT * FROM (SELECT id FROM users) u

-- 중첩 서브쿼리  
SELECT * FROM (
  SELECT * FROM (
    SELECT id FROM users
  ) inner1
) outer1
```

### 2. 별칭 처리 테스트
```sql
-- 다양한 별칭 형식
SELECT * FROM (SELECT id FROM users) AS u
SELECT * FROM (SELECT id FROM users) u
SELECT * FROM (SELECT id FROM users) as u
```

### 3. autoIndent 모드 비교 테스트
- `autoIndent=true` vs `autoIndent=false` 결과 비교
- 서브쿼리 처리 일관성 확인
- 성능 벤치마크

### 4. 복잡한 쿼리 테스트
```sql
-- CTE + 서브쿼리 조합
WITH cte AS (
  SELECT * FROM users WHERE id IN (SELECT user_id FROM orders)
)
SELECT * FROM cte WHERE EXISTS (SELECT 1 FROM profiles p WHERE p.user_id = cte.id)
```
