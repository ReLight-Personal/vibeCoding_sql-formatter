# SQL Formatter Engine Upgrade & Dialect Badge
### 0. Claude

## 개요

자체 구현한 `lib/sql-formatter` 엔진의 미완성 기능을 전면 완성하고,
감지된 SQL 방언을 Input 패널 상단에 배지로 표시하는 UI 개선 작업 기록

## 1. 라이브러리 엔진 개선 (`src/lib/sql-formatter`)

### 1-1. 토크나이저 (`core/tokenizer.ts`)

**버그 수정**
- 공백 토큰이 `isKeyword()` 체크를 타는 로직 오류 수정
- 연속 identifier 문자가 문자 단위로 쪼개지는 현상 → 병합 후처리(`mergeIdentifiers`) 추가

**기능 추가**
- 복합 연산자 우선 매칭 (`<=`, `>=`, `<>`, `!=`, `::`, `||`, `->` 등)
- 백틱 식별자 처리 추가 (MySQL `` `table_name` ``)
- MyBatis 파라미터 토큰 타입 추가: `mybatis_param` (`#{param}`, `${param}`)
- MyBatis XML 태그 토큰 타입 추가: `mybatis_tag` (`<if>`, `<where>`, `<foreach>` 등)
- 키워드 세트 확장 (방언별 키워드 포함)

**토큰 타입 추가** (`types/token.ts`)
```typescript
| 'mybatis_tag'    // <if>, <where>, <foreach> 등
| 'mybatis_param'  // #{param}, ${param}
| 'placeholder'    // 내부 플레이스홀더 (MyBatis 처리용)
```

### 1-2. 파서 (`core/parser.ts`)

기존에 `null` 반환 또는 `parseGenericStatement()`로 위임만 하던 절들을 전면 구현

| 구분 | 변경 전 | 변경 후 |
|------|---------|---------|
| GROUP BY | `return null` (미구현) | 완전 구현 |
| ORDER BY | `return null` (미구현) | 완전 구현 |
| HAVING | 없음 | 신규 구현 |
| JOIN | 없음 | INNER / LEFT / RIGHT / FULL / CROSS / NATURAL 전체 구현 |
| INSERT | generic 위임 | VALUES 다중 행, INSERT INTO SELECT 구현 |
| UPDATE | generic 위임 | SET 절, FROM (PostgreSQL), RETURNING 구현 |
| DELETE | generic 위임 | USING (PostgreSQL), RETURNING 구현 |
| CTE (WITH) | 없음 | 신규 구현 |
| 서브쿼리 | 없음 | 재귀 파싱 구현 |
| UNION / INTERSECT / EXCEPT | 없음 | 신규 구현 |
| CREATE TABLE / VIEW | 없음 | 신규 구현 |
| PL/SQL DECLARE / BEGIN...END | 없음 | EXCEPTION 블록 포함 구현 |

**조건식 파싱 개선**
- `AND` / `OR` 기준으로 `condition_group` 분리
- 괄호 깊이(depth) 추적으로 서브쿼리 내부 키워드 오인식 방지

### 1-3. 포매터 (`core/formatter.ts`)

파서에서 구현된 모든 노드 타입에 대한 포매팅 로직 추가

**MyBatisTemplateHandler 클래스 신규 추가**
- 포매팅 전: MyBatis 태그/파라미터 → 플레이스홀더로 치환
- 포매팅 후: 플레이스홀더 → 원본 태그/파라미터로 복원
- `templateType: 'mybatis'` 옵션 또는 자동 감지로 활성화

**commaPosition 이중 처리 제거**
- 기존: `formatSql.ts`의 `applyCommaLeading()` 후처리 + 라이브러리 내부 처리 중복
- 변경: 라이브러리 내부에서 단일 처리로 통일

### 1-4. 진입점 (`index.ts`)

간편 함수 추가
```typescript
export function formatPostgreSql(sql, options?)  // 신규
export function formatTSql(sql, options?)         // 신규
export function formatMybatisSql(sql, options?)   // 신규
```

## 2. UI 개선 — 감지된 방언 배지 표시

### 변경 파일

#### `src/utils/formatSql.ts`

- 반환 타입을 `string` → `FormatResult`로 확장
- `SqlFormatter` 인스턴스의 `detectDialect()` 호출 추가
- `applyCommaLeading()` 후처리 제거 (라이브러리 내부 처리로 통일)

```typescript
export interface FormatResult {
  sql: string
  detectedDialect: SqlDialect
  detectedDialectLabel: string  // 화면 표시용 라벨
}

const DIALECT_LABEL: Record<SqlDialect, string> = {
  sql:         'Standard SQL',
  plsql:       'PL/SQL',
  mysql:       'MySQL',
  postgresql:  'PostgreSQL',
  transactsql: 'T-SQL',
}
```

#### `src/App.tsx`

- `detectedDialect` 상태 추가 (`useState<string>('')`)
- `runFormat()` 내에서 `FormatResult`로 수신 후 상태 저장
- `EditorContainer`에 `detectedDialect` prop 전달

#### `src/components/EditorContainer.tsx`

- `detectedDialect?: string` prop 추가
- Input용 `EditorPanel`에만 `badge` prop으로 전달 (Output에는 미적용)

#### `src/components/EditorPanel.tsx`

- `badge?: string` prop 추가
- `editor-header` 내 배지 조건부 렌더링

```tsx
{badge && <span className="editor-dialect-badge">{badge}</span>}
```

#### `src/components/EditorPanel.css`

- `editor-header`: `block` → `flex` + `justify-content: space-between`으로 변경
- `editor-dialect-badge` 스타일 신규 추가

```css
.editor-dialect-badge {
  font-size: 0.7rem;
  font-weight: 600;
  padding: 2px 8px;
  border-radius: 9999px;
  background-color: hsl(var(--primary) / 0.12);
  color: hsl(var(--primary));
  border: 1px solid hsl(var(--primary) / 0.3);
  letter-spacing: 0.04em;
  white-space: nowrap;
```

## 3. 변경된 기능 흐름

```
[변경 전]
App.tsx → formatWithRules(sql, rules): string
        → applyCommaLeading() 후처리 (leading일 경우)
        → setOutputSql(result)

[변경 후]
App.tsx → formatWithRules(sql, rules): FormatResult
        → setOutputSql(result.sql)
        → setDetectedDialect(result.detectedDialectLabel)
        → EditorContainer → EditorPanel(badge) 표시
```
