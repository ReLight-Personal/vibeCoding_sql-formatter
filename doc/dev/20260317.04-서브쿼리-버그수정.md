# 서브쿼리 처리 버그 수정

### 작업 도구
- **Claude Sonnet 4.6**

---

## 개요

`subqueryUtils.ts` 분리 이후 실사용 테스트에서 발견된 4가지 버그를 수정.  
비-auto 모드에서 서브쿼리 들여쓰기 누락, `(` 위치 정렬 불일치, DECODE 등 SQL 함수 뒤 키워드 인식 실패, AND 조건 앞 불필요한 빈 줄 삽입이 대상이다.

---

## 발견된 버그 목록

### Bug 1 — 서브쿼리 내부 들여쓰기 미적용

**증상:** 비-auto 모드에서 서브쿼리 내부 줄이 들여쓰기 없이 출력됨.

```sql
-- 기대
SELECT
  (SELECT s.K_D_NM
     FROM HDT s
    WHERE s.D_CD = a.W_CD
  ) AS W_NM
FROM MDT a

-- 실제 (버그)
SELECT
  (SELECT s.K_D_NM
FROM HDT s
WHERE s.D_CD = a.W_CD
) AS W_NM
FROM MDT a
```

**원인:**  
`applySubqueryFormatting`의 `formatInner` 콜백이 서브쿼리 내부 SQL의 들여쓰기 기준(`baseIndent`)을 전달받지 못함.  
콜백 시그니처가 `(inner: string) => string` 으로 고정되어 있어, 복원 시점에 `(` 의 컬럼 위치를 알 수 없었음.

---

### Bug 2 — 서브쿼리 내부 시작 위치가 `(` 기준이 아님

**증상:** 서브쿼리 내부 줄들이 `(` 위치가 아닌 임의의 위치에서 시작됨.

**원인:**  
`restoreSubqueries`가 `closingIndent`(닫는 `)` 기준 들여쓰기)를 계산하여 콜백에 전달하지 않으므로,  
내부 각 줄에 적용할 `baseIndent`를 `formatInner`가 스스로 결정할 수 없었음.  
Bug 1과 동일한 근본 원인.

---

### Bug 3 — DECODE 등 SQL 함수 이후 키워드 인식 실패

**증상:** 컬럼에 `DECODE(...)` 가 포함된 경우, 그 뒤에 오는 `FROM`, `WHERE` 등이 줄바꿈되지 않거나 한 줄로 붙어 출력됨.

```sql
-- 기대
SELECT decode(a.T_CD, 'A', a.A_CD, 'B', a.B_CD, a.T_CD) AS T_CD
  FROM HDT s
 WHERE s.D_CD = a.W_CD

-- 실제 (버그)
SELECT decode(a.T_CD, 'A', a.A_CD, 'B', a.B_CD, a.T_CD) AS T_CD FROM HDT s WHERE s.D_CD = a.W_CD
```

**원인:**  
`splitRespectingParens`에서 괄호 깊이만 추적하고 **문자열 리터럴(`'...'`) 내부를 건너뛰지 않음.**  
`DECODE(col, 'AND', 'value')` 처럼 리터럴 안에 키워드 문자열이 있는 경우, 괄호 깊이 계산이 어긋나  
이후 실제 `FROM` · `WHERE` 위치의 depth 판정이 틀려짐.

---

### Bug 4 — 비-auto 모드에서 AND 위에 빈 줄 추가

**증상:** WHERE 절의 AND 조건 앞에 의도하지 않은 빈 줄이 삽입됨.

```sql
-- 기대
WHERE a.PT = '02459026'
  AND a.O_YMD = '2026-03-13'
  AND a.MND_CD = 'A'

-- 실제 (버그)
WHERE a.PT = '02459026'

AND a.O_YMD = '2026-03-13'

AND a.MND_CD = 'A'
```

**원인:**  
`applySubqueryFormatting`이 `formatAst` 결과 전체에 `presplitByKeywords`를 적용함.  
`AND` · `OR`가 `AUTO_CLAUSE_KEYWORDS`에 포함되어 있어, `formatWhereClause`가 이미 올바르게 분리한 AND 조건 앞에 줄바꿈이 **한 번 더** 삽입됨.

## 수정 내용

### 1. `restoreSubqueries` — `formatInner` 시그니처 확장 (Bug 1 · Bug 2)

`formatInner` 콜백에 `baseIndent` 파라미터를 추가.  
`closingIndent`(닫는 `)` 기준 들여쓰기)에 1단계를 더한 값을 콜백에 전달하여  
서브쿼리 내부 줄이 `(` 위치 기준으로 정렬되도록 수정.

```typescript
// 변경 전
formatInner?: (inner: string) => string

// 변경 후
formatInner?: (inner: string, baseIndent: string) => string
```

`indentBlock(sql, baseIndent)` 헬퍼를 신규 export하여  
각 줄 앞에 `baseIndent`를 일관되게 붙이는 공통 로직을 분리.

```typescript
export function indentBlock(sql: string, baseIndent: string): string {
  return sql
    .split('\n')
    .map(line => (line.trim() ? baseIndent + line.trim() : ''))
    .join('\n')
}
```

---

### 2. `splitRespectingParens` — 문자열 리터럴 보호 (Bug 3)

괄호 깊이 계산 루프에 문자열 리터럴(`'...'`) 스킵 로직 추가.  
`''` 이스케이프(연속 따옴표)도 정상 처리.

```typescript
// 변경 전: 괄호만 추적
for (let i = 0; i < matchStart; i++) {
  if (sql[i] === '(') depth++
  else if (sql[i] === ')') depth--
}

// 변경 후: 문자열 리터럴 내부 건너뜀
let inString = false
for (let i = 0; i < matchStart; i++) {
  const ch = sql[i]
  if (inString) {
    if (ch === "'") {
      if (sql[i + 1] === "'") { i++; continue }  // '' 이스케이프
      inString = false
    }
    continue
  }
  if (ch === "'") { inString = true; continue }
  if (ch === '(') depth++
  else if (ch === ')') depth--
}
```

### 3. `applySubqueryFormatting` — 최상위 `presplitByKeywords` 제거 (Bug 4)

`formatAst` 결과 전체에 `presplitByKeywords`를 적용하는 코드를 제거.  
`presplitByKeywords`는 추출된 서브쿼리 내부(`formatInner` 콜백 안)에만 적용.

```typescript
// 변경 전
private applySubqueryFormatting(sql: string, cfg: FormatterConfig): string {
  const presplit = presplitByKeywords(sql, cfg)   // ← 제거
  const extracted = extractSubqueries(presplit)
  ...
}

// 변경 후
private applySubqueryFormatting(sql: string, cfg: FormatterConfig): string {
  const extracted = extractSubqueries(sql)         // 원본에서 직접 추출
  ...
  // presplitByKeywords는 formatInner 콜백 내부에서만 호출
}
```

`formatInnerSubquery` 헬퍼 메서드를 분리하여 중첩 서브쿼리의 재귀 처리에 `baseIndent`를 일관되게 전달.

---

### 4. `autoIndentFormatter.ts` — 콜백 시그니처 동기화

`restoreSubqueries` 콜백 시그니처 변경에 맞춰 인자 추가.  
auto 모드에서는 `restoreSubqueries`가 전달한 `subBaseIndent`를 `formatBlock`의 기준 들여쓰기로 사용.

```typescript
// 변경 전
(inner) => {
  const normalizedInner = presplitByKeywords(inner, cfg)
  return formatBlock(normalizedInner, cfg, baseIndent)
}

// 변경 후
(inner, subBaseIndent) => {
  const normalizedInner = presplitByKeywords(inner, cfg)
  return formatBlock(normalizedInner, cfg, subBaseIndent)
}
```
