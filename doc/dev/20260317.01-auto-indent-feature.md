# Auto 들여쓰기 기능 개발 로그

### 0. 작업 환경
- **모델**: Claude Sonnet 4.6
- **작업일**: 2026-03-17

---

## 1. 작업 배경

기존 들여쓰기 옵션은 `Space(N칸)` / `Tab` 두 가지만 존재했다.
사용자 요청으로 **키워드 기준 자동 정렬**을 수행하는 `자동(AUTO)` 옵션을 신규 추가.

SQL-Tailor 고유의 정렬 방식으로, 외부 포매터에는 없는 기능이다.

---

## 2. 요구사항

### 초기 요구사항 (1차)

| 항목 | 내용 |
|------|------|
| UI | 규칙 제어 > 들여쓰기 항목에 "자동" 라디오 버튼 추가 |
| Rule 1 | SQL 키워드 문구마다 줄바꿈 |
| Rule 2 | 가장 긴 키워드 길이만큼 나머지 키워드 앞에 우측 정렬 패딩 |
| 구현 위치 | `sql-formatter` core 영역에 전용 파일 생성 |

### 추가 수정 요구사항 (2차 — 버그 픽스)

스크린샷 검토 후 3가지 문제 발견 및 수정 요청:

| # | 문제 | 원인 | 해결 |
|---|------|------|------|
| 1 | SELECT 내 서브쿼리 정렬 미적용 | 줄 단위 처리만 해서 서브쿼리 내부 키워드가 매칭 안 됨 | 서브쿼리 블록을 플레이스홀더로 추출 후 재귀 처리 |
| 2 | AND/OR 키워드 들여쓰기 미적용 | `AUTO_CLAUSE_KEYWORDS` 목록에 AND, OR 누락 | 목록에 AND, OR 추가 |
| 3 | 키워드 뒤 첫 내용이 줄바꿈됨 | 기존 로직이 키워드와 내용을 별도 줄로 분리 | Rule 3 적용: 키워드 뒤에 2칸 공백으로 같은 줄 연결 |

### 추가 수정 요구사항 (3차 — 서브쿼리 정렬 개선)

| 항목 | 내용 |
|------|------|
| 요청 | 서브쿼리 시작 위치를 `(` 다음 위치 기준으로 정렬 |
| 결과 | `(SELECT  col` 처럼 `(` 바로 뒤에서 서브쿼리 내용 시작, 이후 줄은 `(` 다음 열 기준 정렬 |

## 3. 구현 내용

### 3.1 신규 파일

**`src/lib/sql-formatter/core/autoIndentFormatter.ts`**

Auto 들여쓰기 전용 포매터. `applyAutoIndent(sql, cfg)` 함수를 export.

```
전체 흐름:
  applyAutoIndent(sql, cfg)
    └── formatBlock(sql, cfg, baseIndent='')
          ├── extractSubqueries(sql)        서브쿼리 추출 & 플레이스홀더화
          ├── 줄 단위 세그먼트 분류
          │     matchClauseKeyword() 로 키워드 줄 판별
          ├── maxLen 계산 (사용된 키워드 중 최대 길이)
          ├── 줄 재구성
          │     - 키워드 줄: 우측정렬 패딩 + 2칸 공백 + rest
          │     - 비키워드 줄: contentPad(maxLen+2) + rawLine
          └── 서브쿼리 복원 (재귀 formatBlock)
                - '(' 다음 위치를 subqBaseIndent로 계산
                - 첫 줄: before+'('+firstInnerLine
                - 닫는 ')': subqBaseIndent-1 열에 위치
```

**핵심 알고리즘 — 서브쿼리 추출 (`extractSubqueries`)**

```
반복 (안→밖 순서):
  regex /\(([^()]*)\)/ 로 중첩 없는 가장 안쪽 괄호 탐색
  내부에 SELECT 포함 시:
    - '(__SUBQ_N__' 으로 치환 ('(' 를 유지)
    - alias 추출: ) 뒤의 [as] word 패턴, raw length 사용
  반복 종료: 더 이상 매칭 없을 때
```

**핵심 알고리즘 — '(' 기준 정렬**

```
플레이스홀더 복원 시:
  result에서 플레이스홀더가 포함된 줄 탐색
  idx = 플레이스홀더 시작 위치 = '(' 다음 열
  subqBaseIndent = ' '.repeat(idx)
  formatBlock(inner, cfg, subqBaseIndent) 로 재귀

  복원:
    before + firstInnerLine.trimStart()     <- '(' 에 바로 붙음
    ...innerLines[1:]                       <- subqBaseIndent 기준 정렬됨
    closingIndent + ')' + aliasStr          <- '(' 열에 맞춤
```

### 3.2 수정 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/types/formatRules.ts` | `IndentType`에 `'auto'` 추가 |
| `src/lib/sql-formatter/types/config.ts` | `IndentType`에 `'auto'` 추가 (코어 타입 동기화) |
| `src/utils/formatSql.ts` | `indentType === 'auto'` 분기 추가, `applyAutoIndent()` 호출 |
| `src/components/RulePanel.tsx` | 들여쓰기 섹션에 "자동" 라디오 버튼 및 `AUTO` 배지 추가 |
| `src/components/RulePanel.css` | `.auto-badge`, `.auto-desc` 스타일 추가 |

### 3.3 `formatSql.ts` 연동 방식

```typescript
// auto 모드: 내부적으로 'spaces'로 기본 포매팅 후 후처리
if (rules.indentEnabled && rules.indentType === 'auto') {
  const autoCfg: FormatterConfig = { ... }
  formattedSql = applyAutoIndent(formattedSql, autoCfg)
}
```

기본 포매터(SqlFormatter)는 `'spaces'` 모드로 동작하고,
Auto 포매터는 그 출력을 받아 후처리하는 파이프라인 구조.

## 4. 최종 출력 예시

**입력 SQL:**
```sql
select O_CD, (select s.K_D_NM from HDT s where s.D_CD = a.W_CD) as W_NM, a.*
from MDT a
where a.PT = '02459026' and a.O_YMD = '2026-03-13' and a.MND_CD = 'A'
```

**Auto 들여쓰기 출력 (소문자, trailing comma 기준):**
```sql
SELECT  O_CD
        , (SELECT  s.K_D_NM
             FROM  HDT s
            WHERE  s.D_CD = a.W_CD
          )  as W_NM, a.*
  FROM  MDT a
 WHERE  a.PT = '02459026'
   AND  a.O_YMD = '2026-03-13'
   AND  a.MND_CD = 'A'
```

**정렬 기준 (SELECT = 6자가 최장):**

```
키워드    길이  패딩   출력
──────────────────────────
SELECT    6     0     SELECT
FROM      4     2       FROM
WHERE     5     1      WHERE
AND       3     3        AND
```

## 6. 알려진 제한 사항

| 항목 | 내용 |
|------|------|
| 다중 중첩 서브쿼리 | 안→밖 순서 재귀 처리로 정상 동작하나, 3단계 이상 중첩 시 닫는 `)` 정렬이 다소 어긋날 수 있음 |
| CASE WHEN | CASE/WHEN/THEN/ELSE/END는 Auto 키워드 목록 미포함. 추후 추가 예정 |
| 주석 포함 SQL | 인라인 주석(`--`)이 키워드와 같은 줄에 있으면 정렬이 깨질 수 있음 |
