# SQL Formatter 유틸리티 모듈 분리 리팩토링

### 0. Claude
- **Claude Sonnet 4.6 모델 사용**

## 개요

`formatter.ts`에 혼재해 있던 MyBatis 처리·렌더링·방언 감지 로직과
`tokenizer.ts`에 내장되어 있던 키워드 데이터를 독립 유틸리티 파일로 분리.
`core/` 안에 `utils/` 폴더를 신설하여 핵심 파이프라인 파일과 보조 유틸리티를 구분한다.

## 기존 구조의 문제점

### 1. formatter.ts 과부하 (956줄)
- MyBatis 태그 치환·복원 클래스(`MyBatisTemplateHandler`)가 포매팅 로직과 혼재
- 토큰 렌더링·들여쓰기·케이스 변환 등 순수 함수들이 클래스 멤버로 묶여 재사용 불가
- `detectDialect()` 방언 감지가 `SqlFormatter` 클래스 내부에 위치

### 2. 키워드 데이터 접근 불가
- SQL 키워드 목록(`keywords`)이 `SqlTokenizer`의 `private` 필드로 은닉
- tokenizer 외부에서 키워드 확인 필요 시 접근 방법 없음
- `detectDialect()`와 키워드 목록이 서로 다른 파일에 분산

### 3. core/ 폴더 가독성
- 핵심 파이프라인 파일(tokenizer, parser, formatter)과 유틸리티 파일(subqueryUtils)이 동일 레벨에 혼재
- 폴더를 열었을 때 무엇이 메인 모듈인지 직관적으로 파악하기 어려움

### 2. mybatisUtils.ts 분리

`formatter.ts`에 내장되어 있던 MyBatis 처리 전체를 추출.

```typescript
// core/utils/mybatisUtils.ts

export const STRUCTURAL_TAG_RE = /^<\/?(?:where|set)\b/i
export const BLOCK_TAG_RE = /^<\/?(?:if|foreach|choose|...)\b/i

export interface ExtractedTemplate {
  sql: string
  placeholders: Map<string, string>
}

export function hasMybatisMarkers(sql: string): boolean
export class MyBatisTemplateHandler {
  extract(input: string): ExtractedTemplate
  restore(sql: string, placeholders: Map<string, string>): string
}
```

`formatter.ts` 변경:
```typescript
// 기존 — 클래스 내부에 직접 정의
class MyBatisTemplateHandler { ... }  // 약 80줄

// 변경 후 — import로 대체
import { MyBatisTemplateHandler, hasMybatisMarkers } from './utils/mybatisUtils'
```

### 3. renderUtils.ts 분리

`SqlFormatter` 클래스의 `private` 메서드였으나 `this`에 의존하지 않는 순수 함수들을 추출.
함수명은 클래스 외부 노출에 맞게 명확하게 재명명.

| 기존 메서드 | 추출 후 함수명 |
|-------------|---------------|
| `this.applyCase(token, cfg)` | `applyTokenCase(token, cfg)` |
| `this.kw(keyword, cfg)` | `applyKeywordCase(keyword, cfg)` |
| `this.indent(cfg, level)` | `buildIndent(cfg, level)` |
| `this.renderTokens(tokens, cfg)` | `renderTokens(tokens, cfg)` |
| `this.formatColumn(...)` | `formatColumnStr(...)` |

```typescript
// core/utils/renderUtils.ts
import type { SqlToken } from '../../types/token'
import type { FormatterConfig } from '../../types/config'

export function applyTokenCase(token: SqlToken, cfg: FormatterConfig): string
export function applyKeywordCase(keyword: string, cfg: FormatterConfig): string
export function buildIndent(cfg: FormatterConfig, level: number): string
export function renderTokens(tokens: SqlToken[], cfg: FormatterConfig): string
export function formatColumnStr(colStr, cfg, indentLevel, index, total): string
```

### 4. dialectUtils.ts 분리

서로 다른 파일에 분산되어 있던 "SQL 언어 지식"을 한 곳으로 통합.

```typescript
// core/utils/dialectUtils.ts
import type { SqlDialect } from '../../types/config'

// tokenizer.ts의 private keywords Set → 공개 상수로 이동
export const SQL_KEYWORDS = new Set([
  'SELECT', 'FROM', 'WHERE', ... // 방언 공통 + 확장 키워드 전체
])

// formatter.ts의 detectDialect() 메서드 → 독립 함수로 이동
export function detectDialect(sql: string): SqlDialect
```

`tokenizer.ts` 변경:
```typescript
// 기존 — private 필드로 직접 보유
private keywords = new Set([ ... ])  // 약 50줄

isKeyword(value: string): boolean {
  return this.keywords.has(value.toUpperCase())
}

// 변경 후 — import로 대체
import { SQL_KEYWORDS } from './utils/dialectUtils'

isKeyword(value: string): boolean {
  return SQL_KEYWORDS.has(value.toUpperCase())
}
```

`formatter.ts`의 `detectDialect` 처리:
```typescript
// 기존 — SqlFormatter 클래스 멤버로 직접 구현
detectDialect(sql: string): SqlDialect {
  if (...mybatis...) return 'mybatis'
  if (...plsql...)   return 'plsql'
  ...
}

// 변경 후 — dialectUtils에 위임하는 래퍼로 전환 (하위 호환 유지)
import { detectDialect } from './utils/dialectUtils'

detectDialect(sql: string) {
  return detectDialect(sql)
}
```

### 5. subqueryUtils.ts 위치 이동

기존 `core/subqueryUtils.ts` → `core/utils/subqueryUtils.ts`로 이동.
내부 로직 변경 없음. types 참조 경로만 `../types` → `../../types`로 수정.

## import 경로 변경 요약

| 파일 | 변경 내용 |
|------|-----------|
| `formatter.ts` | `'./subqueryUtils'` → `'./utils/subqueryUtils'` 외 3개 경로 추가 |
| `tokenizer.ts` | `'./utils/dialectUtils'` import 추가, `private keywords` 필드 제거 |
| `autoIndentFormatter.ts` | `'./subqueryUtils'` → `'./utils/subqueryUtils'` |
| `utils/*.ts` | `'../types/...'` → `'../../types/...'` |

## 개선 효과

### 1. formatter.ts 경량화
- 956줄 → 700줄 (-27%)
- `SqlFormatter` 클래스가 AST 포매팅 로직에만 집중

### 2. 재사용성 확보
- `renderUtils`의 순수 함수들을 향후 다른 포매터 모듈에서 import 가능
- `SQL_KEYWORDS`를 tokenizer 외부에서도 직접 참조 가능

### 3. 언어 지식 단일화
- 키워드 목록 수정 시 `dialectUtils.ts` 한 파일만 변경
- `detectDialect` 로직과 키워드 데이터가 같은 파일에 공존

### 4. 폴더 가독성
- `core/`를 열면 파이프라인 4개 파일만 노출
- `core/utils/`에서 보조 로직 별도 관리
