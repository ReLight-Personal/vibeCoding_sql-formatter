# SQL Formatter Custom Implementation Migration
### 0. Windsurf
- **SWE-1.5 모델 사용**

## 개요
외부 `sql-formatter` 라이브러리에서 직접 구현한 `lib/sql-formatter`로 마이그레이션한 작업 기록

## 변경 사항

### 1. 주요 파일 변경

#### `src/utils/formatSql.ts`
- **Import 변경**: `sql-formatter` → `../lib/sql-formatter`
- **옵션 구조 수정**: 
  - `language: 'sql'` → `dialect: 'sql'`
  - `useTabs: boolean` → `indentType: 'tabs' | 'spaces'`
  - `commaPosition` 옵션 추가
- **타입 호환성**: `IndentType`, `CommaPosition` 타입 import 및 변환 로직 수정
- **백업**: 변경 전 로직 주석으로 보존

### 2. 파일 정리

#### 삭제된 파일
- `src/utils/test-custom-formatter.ts` - App에서 직접 테스트로 불필요
- `src/utils/customSqlFormatter.ts` - `formatSql.ts`와 기능 중복

### 3. 현재 구조

```
src/
├── utils/
│   └── formatSql.ts          # 메인 포매팅 로직 (lib/sql-formatter 사용)
├── lib/
│   └── sql-formatter/        # 직접 구현한 SQL 포매터
│       ├── index.ts          # 진입점
│       ├── core/
│       │   ├── formatter.ts  # 포매터 핵심 로직
│       │   ├── parser.ts     # SQL 파서
│       │   └── tokenizer.ts  # 토크나이저
│       └── types/
│           ├── config.ts     # 설정 타입
│           └── token.ts      # 토큰 타입
└── App.tsx                   # 앱 메인 (formatWithRules 직접 사용)
```

## 기능 흐름

1. **App.tsx** → `formatWithRules(sql, rules)` 호출
2. **formatSql.ts** → `buildFormatOptions(rules)`로 옵션 변환
3. **formatSql.ts** → `formatSql(sql, options)` 호출
4. **lib/sql-formatter** → 직접 구현된 포매터로 SQL 포매팅
5. **후처리** → `applyCommaLeading()`로 콤마 위치 조정 (leading일 경우)

## 테스트 방법

1. 개발 서버 실행: `npm run dev`
2. 브라우저에서 http://localhost:5173 접속
3. SQL 입력 및 규칙 변경으로 실시간 테스트
