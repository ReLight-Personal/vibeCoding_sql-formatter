# MyBatis 엔진 대응 작업 로그
### 0. Claude
- **Sonnet 4.6 모델 사용**

## 증상

MyBatis 쿼리 입력 시 두 가지 문제 발생.

1. 감지 유형이 `MyBatis`가 아닌 `Standard SQL`로 표시됨
2. 포매팅 결과가 구조 없이 **한 줄로** 출력됨

---

## 원인 분석

### 원인 1 — dialect 감지 누락 (`formatSql.ts` / `config.ts`)

`SqlDialect` 타입에 `'mybatis'`가 없었고, `buildFormatOptions()`가 항상 `dialect: 'sql'`을 하드코딩.  
`detectDialect()` 결과가 UI 표시용으로만 쓰이고 실제 `format()` 호출에는 반영되지 않았음.

### 원인 2 — `templateType` 미전달 (`formatSql.ts`)

`formatWithRules()`가 `detectDialect()`를 호출하지만 그 결과를 `options`에 담지 않아  
`format()` 내부의 MyBatis 처리 분기(`hasMybatisMarkers`)가 실행되지 않았음.

### 원인 3 — `mergeIdentifiers()` 버그 (`tokenizer.ts`)

`<if>name = #{name}` 처럼 태그 직후에 값이 붙어있을 때  
태그 플레이스홀더(`__MYBATIS_TAG_N__`)와 다음 identifier가 하나로 합쳐지는 버그.

```
// 버그 결과 예시
__MYBATIS_TAG_4__name   ← 플레이스홀더 + 'name' merge
```

### 원인 4 — `<where>` / `<set>` 태그 치환 후 파서가 절을 못 찾음 (`formatter.ts`)

`<where>` 태그가 단순 플레이스홀더로 치환되면 파서가 `WHERE` 키워드를 찾지 못해  
WHERE 조건 전체가 FROM 절에 편입되거나 누락됨.

### 원인 5 — `<![CDATA[...]]>` 미처리 (`formatter.ts`)

Mapper XML 파일의 CDATA 섹션을 처리하는 로직이 없어서  
`<![CDATA[`가 `<` `!` `[` `CDATA` `[` 로 쪼개져 토큰화 오류 발생.

### 원인 6 — `renderTokens()` dot 공백 처리 버그 (`formatter.ts`)

`dot(.)` 이후 공백 제거 로직이 keyword(`AS`, `IN` 등)에도 적용됨.

```
a.MD_CDAS MD_CD     → a.MD_CD AS MD_CD  (AS 앞 공백 누락)
a.OR_CL_CDIN (...)  → a.OR_CL_CD IN (...) (IN 앞 공백 누락)
```

괄호·콤마 앞뒤 공백 규칙도 누락되어 `IN ( 'B1' , 'B2' )` 처럼 불필요한 공백 포함.

### 원인 7 — `parseStatement()`가 comment·플레이스홀더를 삼킴 (`parser.ts`) ← **한 줄 출력 핵심**

`/*SQL_ID:...*/` 같은 주석이나 `__MYBATIS_CDATA_OPEN__` 토큰이 파서의 첫 토큰이 되면  
keyword가 아니라는 이유로 `parseGenericStatement()`에 진입.  
세미콜론(`;`)이 없으면 그 뒤의 `SELECT` · `FROM` · `WHERE` 전부를 하나의 generic statement로 묶어버림.  
`formatGenericStatement()`는 `renderTokens()`만 호출하므로 구조 없이 한 줄 출력.

```
[파싱 흐름 — 수정 전]
토큰: [comment] [CDATA_OPEN] [SELECT] [FROM] [WHERE] ...
        ↓  keyword 아님
parseGenericStatement() → 세미콜론 없음 → EOF까지 전부 하나의 노드
        ↓
formatGenericStatement() = renderTokens() → 한 줄 출력
```

### `config.ts`

`SqlDialect` 타입에 `'mybatis'` 추가.

```ts
// 수정 전
export type SqlDialect = 'sql' | 'plsql' | 'mysql' | 'postgresql' | 'transactsql'

// 수정 후
export type SqlDialect = 'sql' | 'plsql' | 'mysql' | 'postgresql' | 'transactsql' | 'mybatis'
```

### `formatSql.ts`

- `DIALECT_LABEL`에 `mybatis: 'MyBatis'` 추가
- `buildFormatOptions()`에 `detectedDialect` 파라미터 추가 → MyBatis 감지 시 `templateType: 'mybatis'` 자동 주입
- `formatWithRules()`에서 `detectDialect()` 결과를 먼저 구한 뒤 `buildFormatOptions`에 전달하도록 순서 조정

```ts
function buildFormatOptions(rules, detectedDialect?) {
  return {
    dialect: 'sql' as const,
    // ...
    templateType: (detectedDialect === 'mybatis' ? 'mybatis' : 'none') as 'mybatis' | 'none',
  }
}

export function formatWithRules(sql, rules): FormatResult {
  const formatter = new SqlFormatter()
  const detectedDialect = formatter.detectDialect(sql)        // 먼저 감지
  const options = buildFormatOptions(rules, detectedDialect)  // 감지 결과 전달
  const formattedSql = formatter.format(sql, options)
  // ...
}
```

### `tokenizer.ts` — `mergeIdentifiers()` 수정

MyBatis 플레이스홀더를 merge 경계로 처리해 다음 identifier와 합쳐지는 버그 수정.

```ts
private isMybatisPlaceholder(value: string): boolean {
  return /^__MYBATIS_(TAG|PARAM)_\d+__$/.test(value)
}

// 플레이스홀더는 merge에서 제외
if (this.isMybatisPlaceholder(token.value)) {
  result.push(token); i++; continue
}
while (
  j < tokens.length &&
  tokens[j].type === 'identifier' &&
  !this.isMybatisPlaceholder(tokens[j].value) &&  // 추가
  tokens[j].position.offset === ...
)
```

### `formatter.ts` — `MyBatisTemplateHandler` 개선

#### CDATA 처리 (`extract`)

`<![CDATA[...]]>`를 open · close 각각 별도 플레이스홀더로 보존.  
내부 SQL만 꺼내 포매팅하고, `restore` 시 원위치에 복원.  
CDATA 전용 카운터(`cdataCounter`)를 별도 사용해 TAG/PARAM 번호 충돌 방지.

```ts
let cdataCounter = 0
sql = sql.replace(/<!\[CDATA\[([\s\S]*?)\]\]>/g, (_m, inner) => {
  const openKey  = `__MYBATIS_CDATA_OPEN_${cdataCounter}__`
  const closeKey = `__MYBATIS_CDATA_CLOSE_${cdataCounter}__`
  cdataCounter++
  placeholders.set(openKey, '<![CDATA[')
  placeholders.set(closeKey, ']]>')
  return `${openKey}\n${inner}\n${closeKey}`
})
```

#### `<where>` / `<set>` sentinel 키워드 삽입 (`extract`)

structural 태그 치환 시 파서가 절을 인식할 수 있도록 sentinel 키워드를 함께 삽입.  
모든 태그 앞뒤에 공백을 추가해 인접 토큰과의 merge 버그 원천 차단.

```
<where>  →  " __MYBATIS_TAG_N__ WHERE "
<set>    →  " __MYBATIS_TAG_N__ SET "
기타      →  " __MYBATIS_TAG_N__ "
```

#### `restore()` 개선

- CDATA_OPEN / CLOSE 플레이스홀더 별도 처리
- CDATA_CLOSE(`]]>`) 복원 시 앞에 줄바꿈 자동 추가

#### `detectDialect()` 수정

```ts
// CDATA, #{}, MyBatis 태그 모두 감지
if (/[#$]\{[^}]*\}|<(?:if|where|set|foreach|...)\b|<!\[CDATA\[/i.test(sql))
  return 'mybatis'
```

#### `renderTokens()` 공백 처리 개선

| 케이스 | 수정 전 | 수정 후 |
|--------|---------|---------|
| dot 다음 keyword (`AS`, `IN` 등) | 공백 제거돼 붙음 | 공백 유지 |
| 연산자 앞 공백 | 미보장 | 앞 공백 확보 후 출력 |
| 여는 괄호 `(` 뒤 | 공백 포함 | 공백 없이 다음 토큰 밀착 |
| 콤마 `,` 앞 | 공백 있음 | 앞 공백 제거, 뒤 공백 추가 |
| comment 토큰 뒤 | 다음 토큰과 붙음 | 뒤에 공백 강제 추가 |

### `parser.ts` — `parseStatement()` 수정 ← **한 줄 출력 핵심 수정**

`comment` 토큰과 MyBatis 플레이스홀더를 단독 노드로 반환.

```ts
private parseStatement(tokens, idx): ParseResult {
  // comment → comment_node로 단독 반환
  if (token.type === 'comment') {
    return { node: { type: 'comment_node', tokens: [token] }, nextIndex: idx + 1 }
  }
  // CDATA / TAG 플레이스홀더 → mybatis_placeholder_node로 단독 반환
  if (token.type === 'identifier' && /^__MYBATIS_(CDATA|TAG)_/.test(token.value)) {
    return { node: { type: 'mybatis_placeholder_node', tokens: [token] }, nextIndex: idx + 1 }
  }
  // 기존 keyword 분기...
}
```

## 최종 처리 흐름

```
입력 (MyBatis SQL)
  │
  ├─ formatWithRules()
  │   ├─ detectDialect()             → 'mybatis' 감지
  │   └─ buildFormatOptions(mybatis) → templateType: 'mybatis' 포함
  │
  └─ formatter.format()
      ├─ extract()
      │   ├─ <![CDATA[...]]>  → OPEN/CLOSE 플레이스홀더 분리
      │   ├─ #{param}          → __MYBATIS_PARAM_N__ 치환
      │   └─ <tag>             → __MYBATIS_TAG_N__ 치환
      │       ├─ <where>       → " TAG WHERE "  (sentinel)
      │       └─ <set>         → " TAG SET "    (sentinel)
      ├─ tokenize() → mergeIdentifiers() 플레이스홀더 merge 제외
      ├─ parse()
      │   ├─ comment_node          (단독)
      │   ├─ mybatis_placeholder_node (단독)
      │   └─ select_statement      (절별 구조 파싱)
      ├─ formatAst() → 절별 포매팅
      └─ restore()
          ├─ CDATA_OPEN  → <![CDATA[
          ├─ CDATA_CLOSE → \n]]>
          ├─ TAG         → 원본 태그 복원
          └─ PARAM       → #{param} 복원
```

## 입출력 비교

**입력**
```sql
/*SQL_ID: com-sds-healthcare-ehr-sup-sd-sdc-dao-SdCdDeInDAO-listFlGr*/
<![CDATA[
   select  /*+ 힌트 */
           a.MD_CD as MD_CD ,  a.MD_NM as MD_NM
     from  SDMT a
    where  a.OR_CL_CD in ('B1','B2','B3','B9')
      and  a.FL_GR_CD = #{flGrCd}
   order by a.MD_CD
    ]]>
```

**수정 전 출력**
```
/*SQL_ID:...*/ __MYBATIS_CDATA_OPEN_0__ select /*+힌트*/ a . MD_CD as MD_CD ...
```

**수정 후 출력**
```sql
/*SQL_ID: com-sds-healthcare-ehr-sup-sd-sdc-dao-SdCdDeInDAO-listFlGr*/
<![CDATA[
SELECT
  /*+ 힌트 */ a.MD_CD AS MD_CD,
  a.MD_NM AS MD_NM
FROM
  SDMT a
WHERE
  a.OR_CL_CD IN ('B1', 'B2', 'B3', 'B9')
  AND a.FL_GR_CD = #{flGrCd}
ORDER BY
  a.MD_CD
]]>
```
