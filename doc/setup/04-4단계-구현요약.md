# [4단계] AI 연동 및 프롬프트 최적화 — 구현 요약

## 목표
- LLM API 연동 및 AI 제안 UX
- API Key 입력·유효성 검사, "AI 도움받기" 버튼, 변경 전/후 미리보기 후 적용 여부 선택

---

## 구현 내용

### 1. API Key 타입 및 저장
- **파일**: `src/types/ai.ts`
  - `AiProvider`: `'openai' | 'anthropic'`
  - `validateApiKeyFormat(provider, key)`: OpenAI는 `sk-` 접두사, Anthropic은 길이 등 간단 검사
- **파일**: `src/utils/storage.ts` 확장
  - `loadAiProvider()` / `saveAiProvider(provider)`
  - `loadAiApiKey(provider)` / `saveAiApiKey(provider, key)` — 프로바이더별로 키 저장 (브라우저 LocalStorage, 보안 유의)

### 2. LLM API 호출 및 프롬프트
- **파일**: `src/utils/aiFormat.ts`
  - `buildRulesDescription(rules)`: 현재 포맷 규칙을 LLM용 설명 문단으로 변환
  - `requestAiFormat(provider, apiKey, sql, rules)`: 프로바이더에 따라 API 호출
    - **OpenAI**: `gpt-4o-mini`, Chat Completions API
    - **Anthropic**: `claude-3-5-sonnet-20241022`, Messages API
  - 시스템 프롬프트: "포맷된 SQL만 반환, 설명·마크다운 없이"
  - 사용자 프롬프트: 적용할 규칙 설명 + 포맷할 SQL

### 3. API Key 설정 UI
- **파일**: `src/components/AiPanel.tsx`, `AiPanel.css`
- **위치**: 사이드바 하단 (규칙 제어, 사용자 정의 템플릿 아래)
- **구성**
  - 프로바이더 선택: OpenAI (GPT) / Anthropic (Claude)
  - API Key: 비밀번호 입력 → **저장** 클릭 시 LocalStorage에 저장, 저장 후 마스킹 표시
  - **다시 입력**: 키 초기화 후 재입력
  - **AI 도움받기** 버튼: 키 저장된 상태에서만 활성화, 클릭 시 로딩 후 API 호출

### 4. AI 결과 미리보기 모달
- **파일**: `src/components/AiPreviewModal.tsx`, `AiPreviewModal.css`
- **표시**: 변경 전(현재 Output 또는 Input) / 변경 후(AI 제안) 좌우(또는 상하) 비교
- **버튼**
  - **취소**: 모달만 닫기
  - **Output에 적용**: AI 제안 문자열을 Output에 넣고 모달 닫기

### 5. 앱 연동
- **파일**: `src/App.tsx`
- **상태**: `aiProvider`, `apiKey`, `keySaved`, `aiLoading`, `aiPreview`
- **흐름**
  1. API Key 저장/초기화 시 storage 반영
  2. "AI 도움받기" 클릭 → `requestAiFormat(provider, apiKey, inputSql, rules)` 호출
  3. 성공 시 `aiPreview = { before, after }` 설정 → 모달 표시
  4. "Output에 적용" 시 `setOutputSql(after)`, 모달 닫기

## 확인 방법

1. **API Key 설정**: 사이드바 하단에서 OpenAI 또는 Anthropic 선택 후 API Key 입력 → **저장** → 마스킹 표시 확인
2. **AI 도움받기**: Input에 SQL 입력 후 **AI 도움받기** 클릭 → 로딩 후 모달에 변경 전/후 표시 확인
3. **적용**: **Output에 적용** 클릭 시 Output 영역에 AI 결과가 들어가는지 확인
4. **취소**: **취소** 클릭 시 모달만 닫히고 Output은 그대로인지 확인
5. **저장 유지**: API Key 저장 후 새로고침해도 키가 유지되는지 확인 (다시 입력 전까지)

## 보안 참고

- API Key는 사용자 브라우저의 LocalStorage에만 저장됩니다.
- 키는 해당 브라우저에서만 사용되며, 외부 서버로 전송되는 것은 OpenAI/Anthropic API 호출 시에만 해당합니다.
- 공용 PC에서는 사용 후 **다시 입력**으로 키를 지우는 것을 권장합니다.
