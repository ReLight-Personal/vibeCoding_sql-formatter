# [3단계] 사용자 정의 템플릿 (Regex/Replace) — 구현 요약

## 목표
- 텍스트·정규식 기반 대체 규칙 추가 및 적용
- 포맷 규칙 + 사용자 정의 규칙을 LocalStorage에 저장하여 새로고침 시 유지

---

## 구현 내용

### 1. 커스텀 규칙 타입
- **파일**: `src/types/customRules.ts`
- **타입**
  - `ReplaceRuleType`: `'text' | 'regex'`
  - `ReplaceRuleItem`: `id`, `type`, `find`, `replace`, `enabled`
- **헬퍼**: `createReplaceRule()` — 새 규칙 생성 시 id 부여

### 2. 대체 규칙 적용 로직
- **파일**: `src/utils/applyReplaceRules.ts`
- **함수**: `applyReplaceRules(sql, rules)`
  - `enabled`인 규칙만 순서대로 적용
  - **텍스트**: `find` 문자열을 `replace`로 일괄 치환
  - **정규식**: `new RegExp(find, 'g')`로 매칭 후 `replace`로 치환 (캡처 그룹 사용 가능)
  - 잘못된 정규식 등 예외 시 해당 규칙만 스킵

### 3. 사용자 정의 템플릿 UI
- **파일**: `src/components/TemplatePanel.tsx`, `TemplatePanel.css`
- **위치**: 왼쪽 사이드바에서 규칙 제어 패널 아래
- **입력 폼**
  - 유형: 텍스트 / 정규식(Regex) 선택
  - 찾을 문자열 (정규식 시 placeholder 예: `\bISNULL\b`)
  - 바꿀 문자열
  - **규칙 추가** 버튼
- **규칙 목록**
  - 적용 순서(위 → 아래)로 표시
  - 각 항목: 사용 여부 체크박스, 유형(Text/Regex), 찾을 문자열 → 바꿀 문자열, 삭제(×) 버튼

### 4. LocalStorage 저장/로드
- **파일**: `src/utils/storage.ts`
- **키**: `sql-tailor-format-rules`, `sql-tailor-custom-rules`
- **기능**
  - `loadFormatRules()` / `saveFormatRules(rules)` — 포맷 규칙 (2단계)
  - `loadCustomRules()` / `saveCustomRules(rules)` — 사용자 정의 대체 규칙
- **초기화**: 앱 마운트 시 저장된 값 로드, 없으면 기본값 사용

### 5. 포맷 파이프라인 연동
- **파일**: `src/App.tsx`
- **흐름**: Input → `formatWithRules(input, rules)` → `applyReplaceRules(result, customRules)` → Output
- **실시간 반영**: `rules` 또는 `customRules` 변경 시 기존처럼 Output 자동 재계산
- **저장**: `rules` / `customRules` 변경 시마다 `useEffect`로 LocalStorage에 저장

### 6. 레이아웃
- **사이드바**: `RulePanel` + `TemplatePanel`을 세로로 묶은 `.sidebar` (너비 280px, 스크롤 가능)

## 확인 방법

1. **텍스트 대체**: 유형 "텍스트", 찾을 문자열 `ISNULL`, 바꿀 문자열 `COALESCE` → 규칙 추가 후 SQL 정렬. Output에서 ISNULL이 COALESCE로 바뀌는지 확인.
2. **정규식**: 유형 "정규식", 찾을 문자열 `\bSELECT\b`, 바꿀 문자열 `SELECT` 등으로 테스트.
3. **저장 유지**: 규칙 추가 후 새로고침. 포맷 규칙·사용자 정의 규칙이 그대로인지 확인.
